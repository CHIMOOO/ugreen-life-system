# 图片尺寸规范（奖品图 / 下午茶商品图）

本系统的**奖品图**和**下午茶商品图**在前台 12 套主题里以统一方式展示，并可点击放大到接近全屏。
生成或准备图片时按本规范出图，既清晰又省体积。**只需关心尺寸与清晰度，不必自己压缩或转格式**——
后端在上传时会自动把图转成体积最小的 WebP（见下「上传」）。

## 它们在界面上怎么显示（来自代码事实）

- **卡片缩略展示**：所有主题里，奖品图/商品图都固定高 **176px**（Tailwind `h-44`），宽度填满卡片，
  `object-cover` **居中裁剪铺满**（超出的部分会被裁掉）。
  - 奖品卡：桌面 3 列（单卡约 320px 宽）、平板 2 列、手机 1 列。
  - 下午茶商品卡：桌面/平板 2 列（单卡约 492px 宽）、手机 1 列。
- **点击放大预览**：最大 `92vw × 90vh`，`object-contain` **完整显示不裁剪**——原图越清晰，放大越好看。
- **后台列表/编辑缩略图**：36–64px 的小图（同一张原图缩小即可，无需单独出小图）。

## 推荐规范（结论）

| 项 | 推荐 |
| --- | --- |
| **宽高比** | **16:9**（与卡片裁剪最贴合）；正方形商品也可 **1:1** |
| **推荐像素** | **1600×900** 或 **1920×1080**（16:9）；正方形 **1200×1200** |
| **最低像素** | 1280×720（再小放大会糊） |
| **上限** | 宽边 ≤ 约 2400px（更大没意义，上传会转 WebP 重新编码） |
| **格式** | 随意给 PNG / JPG / WebP——**后端会自动转 WebP**，你不用压缩或转格式 |

**构图要点**：卡片用 `object-cover` 会裁掉两侧或上下一部分，**重要主体务必居中、四周留安全边距**，
避免文字/主体贴边被裁。放大预览是完整原图，所以原图本身要干净清晰。

## 用 AI 生成图片时的提示词建议

- 奖品图 / 横向商品图：`16:9，1600×900，主体居中，背景简洁干净，预留四周安全边距`。
- 正方形商品图：`1:1，1200×1200，主体居中`。
- 不要在图里加重要文字贴边；不要输出超大尺寸（>2400px 宽）徒增体积。

## 上传

图片本身的录入 / 上传走后端接口，**部署地址（BASE）、鉴权、上传接口**等见 `tea-product-import` skill
（不要假设是 localhost，按部署环境确认）。要点：

- 上传接口 `POST $BASE/api/admin/upload`（multipart，字段名 `file`，限 5MB，需管理员鉴权）。
- 上传成功返回的 url 形如 `/uploads/xxxx.webp`（后端已自动转 WebP；压缩质量由后台「系统设置 → 图片上传」的
  `image_quality` 决定，默认 90）。把该 url 作为奖品/商品的 `image` 字段即可。
