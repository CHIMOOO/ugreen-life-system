# 图片尺寸规范（奖品图 / 下午茶商品图）

本系统的**奖品图**和**下午茶商品图**在前台 12 套主题里以统一方式展示，并可点击放大到接近全屏。
生成或准备图片时按本规范出图，既清晰又省体积。**只需关心尺寸与清晰度，不必自己压缩或转格式**——
后端在上传时会自动把图转成体积最小的 WebP（见下「上传」）。

## 它们在界面上怎么显示（来自代码事实）

- **奖品图 = 锁定 16:9**：所有 12 套主题里，奖品图都用 `aspect-video`（16:9）框，宽度填满卡片、
  `object-cover` **居中裁剪铺满**。**开奖前**的「奖品」栏与**开奖后**「中奖名单」里的缩略图**都是 16:9**，前后一致
  （这是硬约定，别再假设是固定高度）。
  - 奖品卡：桌面 3 列（单卡约 320px 宽 → 显示约 320×180）、平板 2 列、手机 1 列。
- **下午茶商品图**：固定高 **176px**（Tailwind `h-44`），宽度填满卡片、`object-cover` 居中裁剪。
  - 商品卡：桌面/平板 2 列（单卡约 492px 宽）、手机 1 列。
- **点击放大预览**：奖品图 / 商品图都可点开，最大 `92vw × 90vh`，`object-contain` **完整显示不裁剪**——原图越清晰，放大越好看。
- **后台列表/编辑缩略图**：36–64px 的小图（同一张原图缩小即可，无需单独出小图）。

## 推荐规范（结论）

| 项 | 推荐 |
| --- | --- |
| **宽高比** | **奖品图必须 16:9**（卡片本身就是 16:9 框，其它比例会被上下/左右裁掉）；下午茶商品图 **16:9 或 1:1** 皆可 |
| **推荐像素** | **1600×900** 或 **1920×1080**（16:9）；正方形商品图 **1200×1200** |
| **最低像素** | 1280×720（再小放大会糊） |
| **上限** | 宽边 ≤ 约 2400px（更大没意义，上传会转 WebP 重新编码） |
| **格式** | 随意给 PNG / JPG / WebP——**后端会自动转 WebP**，你不用压缩或转格式 |

**构图要点**：卡片用 `object-cover` 会裁掉两侧或上下一部分，**重要主体务必居中、四周留安全边距**，
避免文字/主体贴边被裁。放大预览是完整原图，所以原图本身要干净清晰。

**关于品牌 logo / 商标——允许，且鼓励**：奖品往往就是品牌实物（蓝牙音箱、机械键盘、品牌保温杯、门店券），
下午茶也常来自品牌门店（肯德基、星巴克、喜茶…）。**图里出现真实品牌 logo、包装、商标是正常且加分的**——
它让奖品/商品更可信、更有辨识度，尽管保留或生成。要避免的只是：无关的杂乱水印、二维码、促销角标、
以及贴边被裁的文字，而**不是**品牌标识本身。（此前的规范并没有禁止品牌 logo，这里再明确一次：不要为了"干净"而刻意抹掉 logo。）

## 用 AI 生成图片时的提示词建议

一套好用的**结构**：`主体 + 材质/细节 + 视角 + 光线 + 背景 + 画幅与构图 + 品牌(可选)`。
先说清主体，再补光影和背景，最后固定画幅。下面给可直接套用的模板（把中括号换成你的实物）。

**奖品图（务必 16:9）** —— 目标：一眼看清是什么奖品，质感高级、居中留边。

```
一张 [蓝牙音箱] 的产品图，16:9 横构图（1600×900），主体居中且占画面约 70%，四周留安全边距；
柔和影棚布光 + 轻微投影，材质细节清晰（金属/塑料/织物质感），
背景为干净的浅色渐变或同色系纯色，微景深虚化，无杂物、无贴边文字、无二维码；
如为品牌实物，保留其真实 logo 与包装配色，商业电商详情图质感。
```

- 奖品是**券/虚拟奖品**（奶茶券、话费）：`扁平插画风的 [奶茶] 券/礼品卡，16:9，居中大图标 + 品牌色，干净背景，可含品牌 logo`。
- 奖品是**周边/礼盒**：强调材质与包装，`礼盒微开露出内容物，柔光，16:9`。

**下午茶商品图（16:9 或 1:1）** —— 目标：诱人、新鲜、干净。

```
一张 [提拉米苏] 的美食特写，[16:9 1600×900 / 或 1:1 1200×1200]，45° 俯视或正侧视，
自然柔光 + 浅景深，突出层次与质感（可可粉/奶油/水果的细节），
背景为浅色木纹或素色桌面搭配少量道具，主体居中留边，色彩明快不过曝；
若来自品牌门店（肯德基/星巴克等），保留杯身/包装上的真实品牌 logo。
```

**通用注意**：主体居中、四周留安全边距（`object-cover` 会裁边）；**允许并鼓励真实品牌 logo/商标/包装**；
不要把重要文字/水印放到贴边处；不要输出超大尺寸（>2400px 宽）徒增体积——1600×900/1920×1080 足矣。

## 上传

图片本身的录入 / 上传走后端接口，**部署地址（BASE）、鉴权、上传接口**等见 `tea-product-import` skill
（不要假设是 localhost，按部署环境确认）。要点：

- 上传接口 `POST $BASE/api/admin/upload`（multipart，字段名 `file`，限 5MB，需管理员鉴权）。
- 上传成功返回的 url 形如 `/uploads/xxxx.webp`（后端已自动转 WebP；压缩质量由后台「系统设置 → 图片上传」的
  `image_quality` 决定，默认 90）。把该 url 作为奖品/商品的 `image` 字段即可。
